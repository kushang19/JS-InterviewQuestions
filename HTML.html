Q1. How does a browser render HTML? { 1. Parsing HTML: The browser parses the HTML document to create the DOM (Document Object Model) tree. This involves reading the HTML tags and constructing a tree structure where each node represents an HTML element. 2. Parsing CSS: The browser parses the CSS (both inline and linked stylesheets) to create the CSSOM (CSS Object Model) tree. This tree represents the styles for the different elements in the HTML document. 3. Constructing the Render Tree: The DOM and CSSOM trees are combined to create the Render Tree. This tree contains only the nodes required for rendering the page (e.g., hidden elements like are excluded). 4. Layout: The browser calculates the position and size of each node in the Render Tree. This process is known as "layout" or "reflow". During this step, the browser determines where each element should appear on the screen. 5. Painting: The browser converts each node in the Render Tree into actual pixels on the screen. This involves drawing the visual representation of each element. 6. Compositing: Modern browsers use layers to optimize rendering. The compositing step involves drawing these layers in the correct order to produce the final visual output on the screen. } Q2. What is the purpose of the DOCTYPE declaration? { The DOCTYPE declaration is used to inform the browser about the version of HTML that the web page is written in. It ensures that the browser renders the page in standards mode rather than quirks mode. Standards mode adheres to web standards, while quirks mode attempts to emulate older browser behavior to ensure compatibility with legacy web pages.
